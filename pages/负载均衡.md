	- 轮询
	  id:: 64046554-ae2c-4319-b330-9d094cfbee8b
	  collapsed:: true
		- 原理
			- 假设后端服务性能都一样，以一次循环的方式将请求调度到不同的服务器上
		- 优缺点
			- 实现简单
			- 如果后端服务器性能不一致，性能好的服务器利用率低，无法发挥最大性能
	- 加权轮询
		- 原理
		  collapsed:: true
			- 按服务器性能分配权重，性能好的权重大，性能差的权重小
			- 需要记录当前轮询到的节点
	- 随机法
		- 原理
			- 在转发请求时随机挑选
		- 优缺点和 ((64046554-ae2c-4319-b330-9d094cfbee8b))一致
			- 区别在于不需要记录轮询的节点
	- 加权随机法
	- 最小连接数
		- 原理
			- 将新请求转发给所有服务器中连接数最小的服务器，因为连接数小通常说明该服务器正在处理的请求最少
		- 场景
			- WebSocket
			- HTTP2.0
			- TCP
			- 等长连接业务场景
		- 基于服务器实际压力来均衡负载
	- 最低延迟
		- 原理
			- 将最新的请求转发给延迟最低的服务器
			- 延迟低说明服务器性能好或负载最低
		- 基于服务器实际压力来均衡负载
	- 源地址Hash法
		- 原理
			- 根据请求来源IP，按照某个规则将请求转发到特定的服务器上
		- 优缺点
			- 可以通过人为干预的方式控制流量
			- 通常用于将特定流量转发到特定机器上实现灰度发布
			- 缺点是同一个用户的请求始终在固定的服务器上
